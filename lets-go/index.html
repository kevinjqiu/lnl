Dynamic Typing

  • No Type information until runtime
  • harder to reason about code
  • hard to optimize
  • interpreter overhead
  • GIL

What About JIT?

  • Restricted to a subset of the language
  • Still no type information in code
  • 3rd party libraries

Errors Ain't Exceptions


  • No try/catch
  • Force you to think about error cases
  • panic/recover for true exceptions

Don't Inherit, Embed


  • Observe open-close principle
  • Share data
  • Not changing behaviour

Type-Safe "Duck"

Through Interfaces


type Duck interface {
  Quack()
}

type Duckula struct {}
func (d Duckula) Quack() {...}
type Donald struct {}
func (d Donald) Quack() {...}

func QuackTogether(ducks []Duck) {
  for _, duck := range ducks {
    duck.Quack()
  }
}
